# Mod Structure & File Formats Reference

## Complete Mod Layout

```
MyMod/
  addon.gproj               # REQUIRED: mod metadata, GUID, dependencies
  resourceDatabase.rdb      # Auto-generated by Workbench (GUID-to-path index)
  thumbnail.png             # Workshop thumbnail
  UserMaps.desc             # User map listing
  Scripts/
    Game/                   # EnforceScript source files (.c)
      Components/           # ScriptComponent subclasses
      Entities/             # ScriptEntity subclasses
      Helpers/              # Static helper classes
      Conf/                 # [BaseContainerProps] data classes
      GameCore/
        Modded/             # Files using "modded" keyword
      UI/                   # UI script logic
      Mission/              # Mission/game mode scripts
      Enums/                # Enum definitions
      Systems/              # Game systems
  Prefabs/                  # .et prefab files
    Characters/
    Vehicles/
    Props/
    Items/
    Weapons/
    AI/
      Waypoints/
  Configs/                  # .conf serialized data
    Editor/
    Map/
  UI/
    layouts/                # .layout widget tree files
    Textures/               # .edds textures, .imageset atlases
      Icons/
  Language/                 # Localization
    MyMod_localization.st   # String table source
    MyMod_localization.en_us.conf  # Runtime locale data
  Particles/                # .ptc particle effects
  Sounds/                   # FMOD .bank audio files
  worlds/
    MapName_Layers/
      default.layer         # World entity placements
      test.layer
  Assets/                   # Raw assets (models, textures, audio source)
```

## addon.gproj Format

Bohemia's proprietary text serialization (not JSON/XML):

```
GameProject {
 ID "MyMod"
 GUID "ABCDEF1234567890"
 TITLE "My Mod Display Name"
 Dependencies {
  "58D0FB3206B6F859"
 }
 Configurations {
  GameProjectConfig PC {
  }
  GameProjectConfig XBOX_ONE {
  }
  GameProjectConfig XBOX_SERIES {
  }
  GameProjectConfig PS4 {
  }
  GameProjectConfig PS5 {
  }
  GameProjectConfig HEADLESS {
  }
 }
}
```

### Key Fields

- `ID` - Short identifier, used as namespace prefix
- `GUID` - 16-char hex, globally unique (assigned by Workbench)
- `TITLE` - Human-readable display name
- `Dependencies` - Space-separated GUIDs of required mods
- `"58D0FB3206B6F859"` - Base Arma Reforger game GUID (almost always required)
- `Configurations` - Per-platform settings (PC, XBOX_ONE, XBOX_SERIES, PS4, PS5, HEADLESS)

### With Localization

```
GameProject {
 ID "MyMod"
 GUID "ABCDEF1234567890"
 TITLE "My Mod"
 Dependencies {
  "58D0FB3206B6F859"
 }
 Configurations {
  GameProjectConfig PC {
   WidgetManagerSettings WidgetManagerSettings "{GUID}" {
    StringTables {
     StringTableDefinition "{GUID}" {
      StringTableSource "{GUID}Language/MyMod_localization.st"
      Languages {
       LanguageDefinition "{GUID}" {
        Code "en_us"
        StringTableRuntime "{GUID}Language/MyMod_localization.en_us.conf"
       }
      }
     }
    }
   }
  }
 }
}
```

## Resource References: {GUID}path Pattern

Every resource reference uses `{GUID}relative/path`:

```enforce
"{E552DABF3636C2AD}Prefabs/Groups/OPFOR/Group_USSR_RifleSquad.et"
"{885DA992DEA7C7BC}UI/Textures/Icons/icon_empty_round.edds"
"{06E1B6EBD480C6E0}Prefabs/AI/Waypoints/AIWaypoint_ForcedMove.et"
```

- GUID in braces identifies which addon owns the resource
- Path after GUID is relative to that addon's root
- GUIDs are assigned by Workbench, stored in `resourceDatabase.rdb`
- Load at runtime: `Resource.Load("{GUID}path/to/file.et")`

## File Types

### Scripts (.c)

EnforceScript source files. Must be under `Scripts/Game/` directory. Compiled alphabetically by path.

### Prefabs (.et)

Entity templates using prototype-based inheritance:

```
GenericEntity : "{GUID}Prefabs/Parent.et" {
 ID "INSTANCE_GUID"
 components {
  MyComponent "{COMP_GUID}" {
   m_fSpeed 10
   m_bActive 1
  }
  RigidBody "{COMP_GUID}" {
   Static 1
  }
 }
 coords 1.5 0 3.2
}
```

- Inherit via `:` after type name
- Only override/add properties that differ from parent
- Components identified by class + instance GUID

### Config (.conf)

Serialized data objects (same syntax as .et):

```
SCR_MapMarkerConfig {
 m_aMarkerEntryConfigs {
  SCR_MapMarkerEntryPlaced "{GUID}" {
   m_sMarkerLayout "{GUID}UI/layouts/MapMarker.layout"
  }
 }
}
```

### World Layers (.layer)

Entity instances placed in world coordinates:

```
SCR_SpawnPoint : "{GUID}Prefabs/MP/Spawning/SpawnPoint_USSR.et" {
 coords 906.641 39.009 2620.362
}
SCR_AIWorld : "{GUID}Prefabs/AI/SCR_AIWorld.et" {
 components {
  NavmeshWorldComponent "{GUID}" {
   NavmeshSettings NavmeshWorld "{GUID}" {
    NavmeshFilesConfig BaseNavmeshFilesConfig "{GUID}" {
     NavmeshFile "{GUID}worlds/Map/Navmeshes/map.nmn"
    }
   }
  }
 }
 coords 1036.795 39 2453.779
}
```

### UI Layouts (.layout)

Widget tree definitions:

```
FrameWidgetClass {
 Name "rootFrame"
 {
  TextWidgetClass "{GUID}" {
   Name "Title"
   Slot FrameWidgetSlot "{GUID}" {
    Anchor 0.5 0 0.5 0
    SizeX 200
    SizeY 40
   }
   Text "#STR_MyMod_Title"
   "Font Size" 24
  }
  ImageWidgetClass "{GUID}" {
   Name "Icon"
   Texture "{GUID}UI/Textures/icon.edds"
   Size 32 32
  }
  ButtonWidgetClass "{GUID}" {
   Name "StartButton"
  }
 }
}
```

Widget types: `FrameWidgetClass`, `TextWidgetClass`, `RichTextWidgetClass`, `ImageWidgetClass`, `ButtonWidgetClass`, `ScrollLayoutWidgetClass`, `GridLayoutWidgetClass`, `OverlayWidgetClass`, `SizeLayoutWidgetClass`

### Imageset (.imageset)

Sprite atlas mapping named sprites to pixel regions:

```
ImageSetClass {
 RefSize 284 788
 Textures {
  ImageSetTextureClass "{GUID}" {
   screenSize FHD
   size 284 788
   path "{GUID}UI/Textures/atlas.edds"
  }
 }
 Images {
  ImageSetDefClass my_icon_name {
   Pos 2 2
   Size 54 110
  }
 }
}
```

### String Tables (.st)

CSV-like localization:

```
ID,English,German,French
STR_MyMod_Title,"My Title","Mein Titel","Mon Titre"
STR_MyMod_Msg,"%1 scored %2 points","%1 erzielte %2 Punkte","%1 a marqu%2 points"
```

Usage in script: `WidgetManager.Translate("#STR_MyMod_Title")`
Usage in widgets: Set text to `"#STR_MyMod_Title"` (auto-translated)

## Workbench Tools

| Tool | Purpose |
|------|---------|
| **World Editor** | Place entities, edit terrain, bake navmesh |
| **Resource Manager** | Browse resources, manage GUIDs, import assets |
| **Script Editor** | EnforceScript IDE with autocomplete |
| **Prefab Editor** | Edit .et files, component-based entity authoring |
| **Particle Editor** | Create/edit .ptc particle effects |
| **Sound Editor** | Configure audio resources |
| **Navmesh Editor** | Bake navigation mesh (.nmn) |
| **Terrain Tools** | Height map editing, texture painting |
| **UI Editor** | Create/edit .layout widget trees |
| **String Table Editor** | Edit .st localization |

Launch via Arma Reforger Tools on Steam (separate app).

## Multi-Addon Projects

Large mods use multiple addon.gproj files:

```
MyProject/
  Core/
    addon.gproj    # GUID "AAA...", depends on base game
  Weapons/
    addon.gproj    # GUID "BBB...", depends on Core + base game
  Vehicles/
    addon.gproj    # GUID "CCC...", depends on Core + base game
```

Benefits: Modular loading, independent Workshop entries, clear dependency chains.
