# Arma Reforger Modding - Enforce Script

**Engine**: Enfusion (Arma Reforger)
**Language**: Enforce Script (.c files) - statically-typed, OOP, C#/C++ hybrid syntax
**Base Game GUID**: `"58D0FB3206B6F859"`

## Quick Reference

### Language Basics

```enforce
// Types
int i = 5;
float f = 3.14;
bool b = true;
string s = "hello";
vector v = "1 0 1";       // 3-component float vector

// String formatting uses %1 %2 tokens (1-indexed)
string msg = string.Format("Player %1 has %2 kills", name, kills);

// Printing
Print("message", LogLevel.NORMAL);

// Arrays

nums.Insert(10);
int count = nums.Count();
int val = nums[0];

// Maps

names.Insert(1, "Alpha");
if (names.Contains(1)) { ... }

// Casting (returns null on failure)
SCR_AIGroup group = SCR_AIGroup.Cast(entity);
MyComponent comp = MyComponent.Cast(entity.FindComponent(MyComponent));
```

### Memory Management (Reference Counting)

```enforce
ref MyClass m_obj;          // Strong reference - prevents deletion
autoptr MyClass temp;       // Auto-deleted when scope exits
MyClass weak;               // Weak reference - no ownership
```

**Rules**: Use `ref` for member variables. Use `autoptr` for temporaries. Never store `autoptr` in class members.

### OOP

```enforce
class MyClass : ParentClass {
  private int m_iPrivate;
  protected float m_fProtected;
  static int s_iShared;

  void MyClass() { }           // Constructor
  void ~MyClass() { }          // Destructor

  override void SomeMethod() {
    super.SomeMethod();         // Always call super in overrides
  }
}

sealed class CannotExtend { }  // No subclasses allowed
```

### The `modded` Keyword (Core Modding Mechanism)

```enforce
// Extends existing class WITHOUT editing base files
modded class SCR_BaseGameMode {
  protected int m_iMyModVar = 0;

  override void OnPlayerConnected(int playerId) {
    super.OnPlayerConnected(playerId);  // ALWAYS call super
    m_iMyModVar++;
  }
}
```

**Rules**: Name must match exactly. Always call `super`. Multiple mods chain via `super`. Cannot `modded` sealed classes. Cannot add `[RplProp]`/`[RplRpc]` in modded classes.

## Entity & Component System

### Entity Hierarchy

```
IEntity > GenericEntity > ScriptedGameEntity
  > BaseGameMode > SCR_BaseGameMode
  > ChimeraCharacter > SCR_ChimeraCharacter
  > Vehicle
  > AIAgent > ChimeraAIAgent > SCR_ChimeraAIAgent
  > AIGroup > ChimeraAIGroup > SCR_AIGroup
```

### ScriptComponent Pattern

Every script component requires a companion Class:

```enforce
[ComponentEditorProps(category: "MyMod/Components", description: "My component")]
class MyComponentClass : ScriptComponentClass {}

class MyComponent : ScriptComponent {
  [Attribute("10", UIWidgets.EditBox, desc: "Some value")]
  protected float m_fValue;

  override void OnPostInit(IEntity owner) {
    super.OnPostInit(owner);
    SetEventMask(owner, EntityEvent.FRAME);
  }

  override void EOnFrame(IEntity owner, float timeSlice) {
    // Per-frame logic (only if EntityEvent.FRAME registered)
  }
}
```

### Component Lifecycle Order

1. `EOnInit` - Entity initialized
2. `OnPostInit` - All components ready, safe to query siblings
3. `EOnActivate` - Entity activated in world
4. `EOnFrame` - Each frame (requires `SetEventMask(owner, EntityEvent.FRAME)`)
5. `EOnContact` - Physics contact
6. `EOnDelete` - Cleanup before deletion

### Finding Components

```enforce
MyComponent comp = MyComponent.Cast(entity.FindComponent(MyComponent));
if (!comp) return;  // ALWAYS null-check
```

## Replication (Multiplayer)

### Authority Check

```enforce
if (Replication.IsServer()) { /* server-only logic */ }

RplComponent rpl = RplComponent.Cast(entity.FindComponent(RplComponent));
if (rpl && rpl.IsProxy()) { /* client copy */ }
if (rpl && rpl.IsOwner()) { /* owning client */ }
```

### RplProp - Replicated Properties

```enforce
[RplProp(onRplName: "OnHealthChanged")]
protected int m_iHealth = 100;

void OnHealthChanged() {
  // Called on proxies when value changes
}

void SetHealth(int val) {
  if (!Replication.IsServer()) return;
  m_iHealth = val;
  Replication.BumpMe();  // Force immediate sync
}
```

### RplRpc - Remote Procedure Calls

```enforce
// Server -> All clients
[RplRpc(RplChannel.Reliable, RplRcver.Broadcast)]
protected void RpcDo_PlayEffect(vector pos) {
  // Runs on all clients
}

// Client -> Server
[RplRpc(RplChannel.Reliable, RplRcver.Server)]
protected void RpcAsk_RequestAction(int actionId) {
  // Runs on server only
}

// Invoke via Rpc() wrapper
Rpc(RpcDo_PlayEffect, position);
```

**Naming convention**: `RpcDo_` = server-to-client, `RpcAsk_` = client-to-server.

**RplChannel**: `Reliable` (guaranteed, ordered) | `Unreliable` (fast, can drop)
**RplRcver**: `Server` | `Owner` | `Broadcast`

### JIP (Join-In-Progress)

- `[RplProp]` values auto-sync to joining players
- RPCs fired before join are NOT received by late joiners
- Always use `RplProp` for persistent state, not RPC

## Event System (ScriptInvoker)

```enforce
// Declaration
ref ScriptInvoker m_OnMyEvent = new ScriptInvoker();

// Subscribe
m_OnMyEvent.Insert(MyCallback);

// Unsubscribe
m_OnMyEvent.Remove(MyCallback);

// Fire
m_OnMyEvent.Invoke(param1, param2);

// Typed invoker (preferred for public APIs)
void MyCallback(int playerId, IEntity player);
typedef func MyCallback;
typedef ScriptInvokerBase<MyCallback> MyCallbackInvoker;
```

Built-in shorthands: `ScriptInvokerVoid`, `ScriptInvokerBool`, `ScriptInvokerFloat`, `ScriptInvokerInt`, `ScriptInvokerEntity`

## Enforce Script Rules (Critical)

These are verified compile-time and runtime constraints:

1. **Always null-check** casts and component lookups — Enforce Script silently crashes on null dereference
2. **Always call `super`** in `modded` class overrides
3. **No multi-line string literal concatenation** — `"foo" "bar"` is invalid; must be one string per line
4. **`string.Format` max 9 params** (%1-%9) — split into multiple `Format` + concatenation if needed
5. **No `ref` for entity classes** — `SCR_AIGroup`, `IEntity`, `SCR_ChimeraCharacter` etc. disallow strong refs. Use plain (weak) references for entities
6. **Use `ref` only for non-entity managed classes** — arrays, custom classes, ScriptInvokers
7. **`autoptr` trap** — never store autoptr in class members; object dies when scope exits
8. **RplProp only in original class** — cannot add in `modded` classes
9. **AI groups need init time** — wait 1-2 seconds after spawn before assigning waypoints
10. **Server-only state changes** — game logic runs on server, effects run anywhere
11. **JIP safety** — use RplProp for state, not one-shot RPCs
12. **Never use `EOnFrame` without need** — prefer `CallLater` or event-driven patterns

### Non-existent APIs (confirmed compile errors)

Do NOT use these — they do not exist in Enforce Script:
- `AIWorld.GetAIGroupCount()` / `AIWorld.GetAIGroup(int)` — no indexed group access
- `World.QueryEntitiesByClassName()` — does not exist
- Use `QueryEntitiesBySphere()` with cast-filtering in callback instead
